# Stageworth

Your theatre life, all in one place. Log every show, rank your favorites, plan trips with friends, and stay on top of what's playing.

**iOS:** [Download on the App Store](https://apps.apple.com/us/app/stageworth/id6761304800)
**Android:** Beta only — see [stageworth.vercel.app](https://stageworth.vercel.app)
**Website:** <https://stageworth.vercel.app>

## Repo layout

| Path        | What                                                                                       |
| ----------- | ------------------------------------------------------------------------------------------ |
| `src/`      | Expo / React Native app (TypeScript, expo-router, React 19).                               |
| `convex/`   | Convex backend — queries, mutations, actions, scheduled jobs (`crons.ts`), HTTP handlers.  |
| `website/`  | Next.js marketing site + `/admin` review tools (deployed to Vercel).                       |
| `bot/`      | `theatre-news-bot` — long-running Bun process that ingests RSS into Convex via `/bot/ingest`. |
| `docs/`     | Architecture notes, plans, and the agent handoff doc.                                      |
| `ios/` `android/` | Native projects, regenerated by `expo prebuild` — usually you can ignore them.      |

## Getting started

Requires [Bun](https://bun.sh), the Expo CLI, and a Convex account / project linked.

```bash
bun install                      # repo root
cd website && bun install        # website has its own lockfile
```

Then, from the repo root:

```bash
bun run dev:convex               # start Convex dev server
bun run dev:expo                 # start Expo on a simulator / dev client
# or
bun run dev                      # both, in the same terminal
```

The website dev server lives in `website/`:

```bash
cd website && bun run dev
```

## Quality gates

```bash
bun run lint                     # ESLint (matches CI)
bunx tsc --noEmit                # typecheck (matches CI)
bun run ci                       # lint + typecheck for app + website
bun run ci:full                  # also runs `next build` for the website
```

CI mirrors these on every PR via `.github/workflows/ci.yml`.

## Releases

See [`docs/AGENT_HANDOFF.md`](docs/AGENT_HANDOFF.md#distribution--release-status) for the full release process. Quick reference:

- **JS/TS-only changes:** publish via `bun run ota:production` (or the **Mobile OTA Update** workflow). Reaches every installed build whose `version` in `app.config.js` matches.
- **Native changes** (new dep, plugin tweak, permission change, SDK upgrade): bump `version` in `app.config.js`, then trigger the **Mobile Store Candidate** workflow (or run `eas build` → `eas submit` locally).
- **Convex changes:** deploy via `bunx convex deploy`. Hits *every* live client immediately — keep them backwards-compatible with the oldest in-the-wild build.
